部署基于Docker的graphite

1. 部署

1.1. 获取CentOS基础镜像

docker pull centos
docker run -it --name graphite --net=host /bin/bash

1.2. 部署graphite基础组件

安装基础环境

yum -y install httpd gcc gcc-c++ git pycairo mod_wsgi epel-release net-tools which telnet iproute 
yum -y install python-pip python-devel blas-devel lapack-devel libffi-devel

获取并安装graphite-web和carbon两大组件

cd /usr/local/src
git clone https://github.com/graphite-project/graphite-web.git
git clone https://github.com/graphite-project/carbon.git
 
pip install -r /usr/local/src/graphite-web/requirements.txt
 
cd /usr/local/src/carbon/
python setup.py install
 
cd /usr/local/src/graphite-web/
python setup.py install

2. 配置

2.1. graphite配置

生成默认配置文件

cp -v /opt/graphite/conf/carbon.conf.example /opt/graphite/conf/carbon.conf
cp -v /opt/graphite/conf/storage-schemas.conf.example /opt/graphite/conf/storage-schemas.conf
cp -v /opt/graphite/conf/storage-aggregation.conf.example /opt/graphite/conf/storage-aggregation.conf
cp -v /opt/graphite/conf/relay-rules.conf.example /opt/graphite/conf/relay-rules.conf
cp -v /opt/graphite/webapp/graphite/local_settings.py.example /opt/graphite/webapp/graphite/local_settings.py
cp -v /opt/graphite/conf/graphite.wsgi.example /opt/graphite/conf/graphite.wsgi

生成apache配置文件

cp -v /opt/graphite/examples/example-graphite-vhost.conf /etc/httpd/conf.d/graphite.conf

如果不是在Docker中而是在裸机上安装，做如下拷贝通过systemctl restart carbon-cache进行服务的起停

cp -v /usr/local/src/carbon/distro/redhat/init.d/carbon-* /etc/init.d/
chmod +x /etc/init.d/carbon-*

2.2. 数据库配置

2.2.1. 使用MySQL数据库

数据库如果用默认配置，则用的是sqlite3，下面是默认配置

$ cat /opt/graphite/webapp/graphite/settings.py

if DATABASES is None:
    DATABASES = {
      'default': {
        'NAME': join(STORAGE_DIR, 'graphite.db'),
        'ENGINE': 'django.db.backends.sqlite3',
        'USER': '',
        'PASSWORD': '',
        'HOST': '',
        'PORT': '',
      },
    }

现在我们用mysql，配置数据库权限如下

$ /opt/graphite/webapp/graphite/settings.py

DATABASES = None
DATABASES = {
    'default': {
    'ENGINE': 'django.db.backends.mysql',    # Add 'postgresql_psycopg2', 'mysql',   'sqlite3' or 'oracle'.
    'NAME': 'graphite',                      # Or path to database   file if using sqlite3.
    'USER': 'graphite',                      # Not used with   sqlite3.
    'PASSWORD': 'graphite',                  # Not used with sqlite3.
    'HOST': '10.17.12.20',                # Set to empty string for localhost. Not used with sqlite3.
    'PORT': '3306',                          # Set to empty string for default. Not used with sqlite3.
    }
}

2.2.2. 初始化数据库

创建MySQL表

create database graphite;
GRANT ALL ON graphite.* TO 'graphite'@'%' IDENTIFIED BY 'graphite';
GRANT ALL ON graphite.* TO 'graphite'@'localhost' IDENTIFIED BY 'graphite';
GRANT ALL ON graphite.* TO 'graphite'@'127.0.0.1' IDENTIFIED BY 'graphite';
flush privileges;

通过参数--settings=graphite.settings，用上面配置文件/opt/graphite/webapp/graphite/settings.py初始化数据库 

#create database
cd /opt/graphite
PYTHONPATH=/opt/graphite/webapp/ django-admin.py migrate --settings=graphite.settings

2.2.3. 错误处理

很多教程中使用的如下命令，执行数据库初始化，报错： 

$ PYTHONPATH=/opt/graphite/webapp/ django-admin.py syncdb --settings=graphite.settings
Unknown command: 'syncdb'
Type 'django-admin.py help' for usage.

在stackoverflow上找到原因和解决方法：

syncdb is deprecated because of the migration system.
Now you can log your changes using makemigrations. This transforms your model changes into python code to make them deployable to another databases.
After you created the migrations you have to apply them: migrate.
So instead of using syncdb you should use makemigrations and then migrate.

简言之：在Django 1.9及未来的版本种使用migrate代替syscdb，而当前版本为： 

$ /usr/bin/django-admin.py version
1.11.9

3. 启动

在基于CentOS7.4的容器内启动

systemctl start carbon-cache 
systemctl start httpd

报错

Failed to get D-Bus connection: Operation not permitted
  • 第一种解决方案：
    这个的原因是因为dbus-daemon没能启动。其实systemctl并不是不可以使用。将你的CMD或者entrypoint设置为/usr/sbin/init即可。会自动将dbus等服务启动起来。然后就可以使用systemctl了。
docker run --privileged -it --name graphite --net=host /usr/sbin/init
  • 第二种解决方案：
    更改启动模式，不用systemctl的方式启动
/usr/bin/python /opt/graphite/bin/carbon-cache.py --instance=a start
/usr/sbin/httpd -D FOREGROUND

后续我们使用Dockerfile制作镜像的时候也是使用的这种方式

carbon-cache安装后开启2003和2004端口，httpd默认使用80端口，安装成功的话通过浏览器访问会出现graphite原始的dashboard

如果出现5xx相关错误，可能是前面忘记设置目录的apache用户权限。

4. 打包Docker镜像

4.1. 设置挂载目录

为方便调试配置和防止容器被数据占满，我们把httpd的配置文件和graphite的存储目录挂载到宿主机

#在宿主机操作
mkdir -vp /data/graphite
mkdir -vp /data/graphite/build-docker-image
docker cp centos:/opt/graphite/storage/ /data/graphite/
docker cp centos:/etc/httpd/ /data/graphite/
mkdir -vp /data/graphite/httpd/logs
chmod 777 /data/graphite/httpd/logs
ln -s /usr/lib64/httpd/modules modules

4.2. Dockerfile

上面基本的部署和配置已经完成，在启动正常的情况下，打包成一个基础镜像， 暂定tag为1.0的镜像。
如果再有对配置的调整可以通过Dockerfile的方式，基于1.0镜像进行调整，如下： 

$ pwd
/data/graphite/build-docker-image

$ cat Dockerfile
FROM registry.your-comp-name.com/projects/graphite:1.0
ADD init.sh /init.sh
RUN unlink /etc/localtime
RUN ln -s /usr/share/zoneinfo/Asia/Shanghai /etc/localtime
RUN sed -i "s/#TIME_ZONE = 'America\/Los_Angeles'/TIME_ZONE = 'Asia\/Shanghai'/" /opt/graphite/webapp/graphite/local_settings.py
RUN sed -i "s/SECRET_KEY = 'UNSAFE_DEFAULT'/SECRET_KEY = 'xxxxxx'/" /opt/graphite/webapp/graphite/settings.py
ENTRYPOINT /bin/bash /init.sh

4.3. 容器初始化脚本

初始化脚本主要是启动carbon-cache和httpd进程。
另外，把一些容易出问题的地方直接放到了容器初始化脚本中。 

$ cat init.sh

#!/bin/bash
chown -R apache:apache /opt/graphite/storage/
chown -R apache:apache /opt/graphite/static/
chown -R apache:apache /opt/graphite/webapp/
chmod 777 /etc/httpd/logs -R

echo "start carbon-cache ... "
/usr/bin/python /opt/graphite/bin/carbon-cache.py --instance=a start
echo $?
ps aux ww f
echo "start httpd ... "
[[ -d "/run/httpd" ]] && rm -vrf /run/httpd
mkdir -vp /run/httpd
/usr/sbin/httpd -D FOREGROUND

4.4. 基于Dockerfile构建镜像

cd /data/graphite/build-docker-image
docker build . -t registry.your-comp-name.com/projects/graphite:1.1

4.5. 启动新镜像

docker rm -f graphite
docker run -d --name graphite --net=host -v /data/graphite/storage:/opt/graphite/storage -v /data/graphite/httpd:/etc/httpd registry.your-comp-name.com/projects/graphite:1.1

4.6. 问题处理

apache启动

可以使用docker run正常启动并提供服务，但是无法通过docker restart graphite重启，Apache日志报错如下： 

[Mon Jan 08 18:17:23.662296 2018] [auth_digest:error] [pid 21] (17)File exists: AH01762: Failed to create shared memory segment on file /run/httpd/authdigest_shm.21
[Mon Jan 08 18:17:23.662316 2018] [auth_digest:error] [pid 21] (17)File exists: AH01760: failed to initialize shm - all nonce-count checking, one-time nonces, and MD5-sess algorithm disabled

问题原因
https://bugzilla.redhat.com/show_bug.cgi?id=1215667

Cannot start httpd in container because of missing /run/httpd

解决，在entrypoint的容器初始化脚本中增加如下: 

$ cat init.sh

[[ -d "/run/httpd" ]] && rm -vrf /run/httpd
mkdir -vp /run/httpd
/usr/sbin/httpd -D FOREGROUND

调整数据保存周期

cat storage-schemas.conf

[default_1min_for_1day]
pattern = .*
#retentions = 60s:7d,300s:30d,1500s:90d
retentions = 60s:1d,5m:7d,1h:30d,1d:1y,1w:5y

5. 安全认证

用Apache Basic Authentication对graphite页面进行安全认证

生成密码文件

htpasswd -b -c pwdfile usernamexxx passwordxxx
mv pwdfile /etc/httpd/

认证配置

<Directory "/opt/graphite/static/">
        Options Indexes FollowSymLinks MultiViews
        AllowOverride None
        Order allow,deny
        Allow from all
        AuthType Basic
        AuthName "Hello,My Friend."
        AuthBasicProvider file
        AuthUserFile /etc/httpd/pwdfile
        Require valid-user
</Directory>
