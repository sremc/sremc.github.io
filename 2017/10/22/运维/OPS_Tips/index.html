cpupower


#设置cpu频率运行模式
sudo cpupower frequency-set -g powersave
sudo cpupower frequency-set -g performance
sudo cpupower frequency-set -g ondemand
#grep cpupower /etc/rc.local || sudo echo "/usr/bin/cpupower frequency-set -g performance" >> /etc/rc.local

#查看cpu0的策略

#查看所有cpu的策略
cpupower -c all frequency-info
cpupower -c all frequency-info --policy
cat /sys/devices/system/cpu/cpu*/cpufreq/scaling_governor 
cat /sys/devices/system/cpu/cpu*/cpufreq/scaling_governor|grep -c performance

cpupower可以调整的级别描述如下：

调速器(governor) 描述
ondemand 按需快速动态调整CPU频率， 一有cpu计算量的任务，就会立即达到最大频率运行，等执行完毕就立即回到最低频率（阙值为 95%）
performance 运行于最大频率
conservative 按需快速动态调整CPU频率， 一有cpu计算量的任务，就会立即达到最大频率运行，等执行完毕就立即回到最低频率（阙值为 75%）
powersave 运行于最小频率
userspace 运行于用户指定的频率

根据实际硬件，以下的调速器可能被默认启用：

  • ondemand ：AMD 及旧款 Intel CPU。
  • powersave ：Intel Sandy Bridge 和更新的CPU。

ethtool

ethtool -s eth0 speed 1000 duplex full autoneg off

若要eth0启动时设置这些参数, 修改文件/etc/sysconfig/network-scripts/ifcfg-eth0 ，添加如下一行: 

ETHTOOL_OPTS="speed 100 duplex full autoneg off"

Linux - mtr

mtr 的功能是检查在目的地址有丢包的情况下，查出具体在哪一跳丢包，

-n 不解析主机名
-i 发包的间隔
-c 发包的次数

mtr -n -i 1 -c 100 x.x.x.x

没有动态显示，最后显示探测报告

mtr -rw x.x.x.x

删除大量小文件

1、建立一个空目录  
mkdir -p /tmp/rsync_blank  
2、确立需要清空的目标目录  
/data/ooxx  
3、使用rsync同步删除（注意目录后面的“/”），整体效率会快一个数量级的样子。  
rsync --delete-before -a -H -v --progress --stats /tmp/rsync_blank/ /data/ooxx/  
选项说明：  
–delete-before 接收者在传输之前进行删除操作  
–progress 在传输时显示传输过程  
-a 归档模式，表示以递归方式传输文件，并保持所有文件属性  
-H 保持硬连接的文件  
-v 详细输出模式  
-stats 给出某些文件的传输状态    
一般我们不需要显示进度，使用以下命令即可  
rsync --delete-before -a -H /tmp/rsync_blank/ /data/ooxx/

getopts 获取多个参数

#!/bin/bash  
[[ $# -eq 0 ]] && echo paramaters missing. && exit

while getopts a:b:c:d: OPTS;do  
case $OPTS in  
    a)aa=$OPTARG  
    echo "option is $OPTS,value is $aa"  
    ;;  
    b)bb=$OPTARG  
    echo "option is $OPTS,value is $bb"  
    ;;  
    c)cc=$OPTARG  
    echo "option is $OPTS,value is $cc"  
    ;;  
    d)dd=$OPTARG  
    echo "option is $OPTS,value is $dd"  
    ;;  
esac  
done

echo a=$aa  
echo b=$bb  
echo c=$cc  
echo d=$dd    

#执行结果如下：
sh test-getopts.sh -a 1 -b 2 -c 3 -d 4
option is a,value is 1
option is b,value is 2
option is c,value is 3
option is d,value is 4
a=1
b=2
c=3
d=4

说明：如getopts ahfvc: option表明选项a、h、f、v可以不加实际值进行传递，而选项c必须取值。使用选项取值时，必须使用变量OPTARG保存该值。

HTTP Header - Cache-Control

Cache-Control : no-cache 的时候 强制每次请求直接发送给源服务器，而不经过本地缓存版本的校验。  
1、适用严格要求使用最新数据 的应用（不惜牺牲使用缓存的所有好处）；  
2、它也用来在客户端发现其缓存中拷贝不可用或过期时，对拷贝进行强制刷新。  
  
Cache-Control : max-age>0 的时候直接从游览器缓存中提取  
  
Cache-Control : max-age<=0 的时候向server发送http请求确认 ,该资源是否有修改  
有修改的话 返回200 ,无修改的话 返回304.

echo要变换颜色的时候，要使用参数-e
格式: echo -e "\033[字背景颜色;字体颜色m字符串\033[0m"

提示符PS1

常用的几个选项参照如下：
/d ：代表日期，格式为weekday month date，例如：”Mon Aug 1”
/H ：完整的主机名称。例如：我的机器名称为：fc8.RedHat，则这个名称就是fc8.RedHat
/h ：仅取主机的第一个名字，如上例，则为fc4，.RedHat则被省略
/t ：显示时间为24小时格式，如：HH：MM：SS
/T ：显示时间为12小时格式
/A ：显示时间为24小时格式：HH：MM
/u ：当前用户的账号名称
/v ：BASH的版本信息
/w ：完整的工作目录名称。家目录会以 ~代替
/W ：利用basename取得工作目录名称，所以只会列出最后一个目录
/# ：下达的第几个命令
/$ ：提示字符，假如是root时，提示符为：# ，普通用户则为：$

完整组合

export PS1='\t \[\e[31;1m\]\u\[\e[0m\]@\[\e[32;1m\]\H\[\e[0m\]:\[\e[34;1m\]\w\[\e[0m\][\#]\\$ '
export PS1="\t \u@10.13.3.241:\w[\#]\\$ "
