CephFS分布式文件系统

1. 部署CephFS分布式文件系统

1.1. 环境说明

Ceph官方文档中建议通过自带的ceph-deploy通过网络进行在线安装，比如CentOS系统就基于yum生态安装，但是考虑到生产环境可能没有公网连接，我们采用docker方式来安装部署CephFS，而且这样做更容易对版本和各个组件进行细致的把控。

为了更清晰的掌握部署环节，现对各个角色规划如下（主机名含义忽略）：

主机名 IP地址 角色 序号
server 172.16.149.129 mon,osd,mds,mgr ceph-1
agent 172.16.149.130 mon,osd,mds ceph-2
server2 172.16.149.131 mon,osd,mds ceph-3

1.2. 准备工作

  • 下载镜像并推送到内网临时私有仓库，方便在每台ceph节点上快速启动docker容器
docker pull registry.xxx.com/ceph/daemon:latest
docker tag registry.xxx.com/ceph/daemon:latest ${harbor-server}/ceph/daemon:latest
docker push ${harbor-server}/ceph/daemon:latest
  • 在每个ceph节点上设置主机名和ip地址的映射（如果内网没有DNS服务）
$ cat /etc/hosts
172.16.149.129 server ceph-1
172.16.149.130 agent ceph-2
172.16.149.131 server2 ceph-3
  • 设置时间同步
$ crontab -l
*/10 * * * * /sbin/ntpdate private-time-server-ip; hwclock -w
  • 硬件环境
    CephFS存储节点需要准备一块单独的磁盘，用于存放数据

1.3. 部署安装

1.3.1. 在ceph-1上启动monitor，在镜像的参数中是mon

docker run -d \
    --name mon \
    --net=host \
    -v /etc/ceph:/etc/ceph \
    -v /var/lib/ceph/:/var/lib/ceph/ \
    -e MON_IP=172.16.149.129 \
    -e CEPH_PUBLIC_NETWORK=172.16.149.0/24 \
    ${harbor-server}/ceph/daemon:latest mon

1.3.2. 拷贝配置文件到另外2台节点

scp -r /var/lib/ceph/bootstrap* ceph-2:/var/lib/ceph/
scp -r /var/lib/ceph/bootstrap* ceph-3:/var/lib/ceph/
scp -r /etc/ceph/* ceph-2:/etc/ceph/
scp -r /etc/ceph/* ceph-3:/etc/ceph/

1.3.3. 分别在ceph-2和ceph-3上启动monitor

#ceph-2
docker run -d \
    --name mon \
    --net=host \
    -v /etc/ceph:/etc/ceph \
    -v /var/lib/ceph/:/var/lib/ceph/ \
    -e MON_IP=172.16.149.130 \
    -e CEPH_PUBLIC_NETWORK=172.16.149.0/24 \
    ${harbor-server}/ceph/daemon:latest mon

#ceph-3
docker run -d \
    --name mon \
    --net=host \
    -v /etc/ceph:/etc/ceph \
    -v /var/lib/ceph/:/var/lib/ceph/ \
    -e MON_IP=172.16.149.131 \
    -e CEPH_PUBLIC_NETWORK=172.16.149.0/24 \
    ${harbor-server}/ceph/daemon:latest mon

1.3.4. 至此，monitor组件在集群中已经启动完毕，检测是否启动成功

$ docker exec mon ceph mon stat

如上所示，三台节点已经启动成功

1.3.6. 部署启动OSD

在每个ceph节点上执行如下命令，创建OSD：

docker run -d \
    --name osd \
    --net=host \
    -v /etc/ceph:/etc/ceph \
    -v /var/lib/ceph/:/var/lib/ceph/ \
    -v /dev/:/dev/ \
    -v /ceph-data:/var/lib/ceph/osd \
    --privileged=true \
    ${harbor-server}/ceph/daemon:latest osd_directory

1.3.7. 部署启动MDS

在每个ceph节点上执行如下命令，创建MDS：

docker run -d \
    --name mds \
    --net=host \
    -v /etc/ceph:/etc/ceph \
    -v /var/lib/ceph/:/var/lib/ceph/ \
    -e CEPHFS_CREATE=1 \
    172.16.149.129/ceph/daemon:latest mds

1.3.8. 部署启动MGR

在ceph-1上启动即可

docker run -d \
    --name mgr \
    --net=host \
    -v /etc/ceph:/etc/ceph \
    -v /var/lib/ceph/:/var/lib/ceph/ \
    172.16.149.129/ceph/daemon:latest mgr

  1. 设置默认启动内核：
$ awk -F\' '$1=="menuentry " {print $2}' /etc/grub2.cfg

3. ceph进阶

相关概念：pool，PG，OSD，对象

  • Pool是存储对象的逻辑分区，它规定了数据冗余的类型和对应的副本分布策略；支持两种类型：副本（replicated）和 纠删码（ Erasure Code）；目前我们公司内部使用的Pool都是副本类型（3副本）；

  • PG（ placement group）是一个放置策略组，它是对象的集合，该集合里的所有对象都具有相同的放置策略；简单点说就是相同PG内的对象都会放到相同的硬盘上； PG是 ceph的核心概念， 服务端数据均衡和恢复的最小粒度就是PG；

  • OSD是负责物理存储的进程，一般配置成和磁盘一一对应，一块磁盘启动一个OSD进程；

下面这张图形象的描绘了它们之间的关系：
一个Pool里有很多PG，
一个PG里包含一堆对象；一个对象只能属于一个PG；
PG有主从之分，一个PG分布在不同的OSD上（针对三副本类型）

在任意节点方便管理设置alias

alias ceph='docker exec mon ceph'

3.1. 存储池

查询池操作

#列出ceph存储池
$ ceph osd lspools
1 cephfs_data,2 cephfs_metadata,

#创建pool
$ ceph osd pool create userFeedPool 64
pool 'userFeedPool' created
$ ceph osd lspools
1 cephfs_data,2 cephfs_metadata,3 userFeedPool,

#pool size 用于指定当前存放在对应池中的文件实际保存的文件副本数量, 常见为 3
#获取存储池的副本数
$ ceph osd pool get userFeedPool size
size: 3

#设置存储池的副本数
$ ceph osd pool get userFeedPool pg_num
pg_num: 64

#获取存储池配额
$ ceph osd pool get-quota userFeedPool
quotas for pool 'userFeedPool':
  max objects: N/A
  max bytes  : N/A

删除存储池

#http://blog.csdn.net/signmem/article/details/78594340
#http://ceph.com/pgcalc/
#直接删除pool会报错
$ ceph osd pool delete userFeedPool userFeedPool --yes-i-really-really-mean-it
Error EPERM: pool deletion is disabled; you must first set the mon_allow_pool_delete config option to true before you can destroy a pool
#以上是 luminous 版本的新特性, 你需要配置 mon_allow_pool_delete 配置后才允许对 pool 进行删除

#查询当前配置：false
$ ceph daemon /var/run/ceph/ceph-mon.$(hostname -s).asok config show | grep mon_allow_pool_delete
    "mon_allow_pool_delete": "false",
$ ceph --admin-daemon /var/run/ceph/ceph-mon.server.asok config show | grep mon_allow_pool_delete
    "mon_allow_pool_delete": "false",

#获得ceph mon socket 位置
$ ceph-conf --name mon.$(hostname -s) --show-config-value admin_socket
/var/run/ceph/ceph-mon.server.asok

#修改配置为：true
$ ceph tell mon.server injectargs '--mon_allow_pool_delete=true'
injectargs:mon_allow_pool_delete = 'true' (not observed, change may require restart)

#查询修改后的结果
$ ceph daemon /var/run/ceph/ceph-mon.$(hostname -s).asok config show | grep mon_allow_pool_delete
    "mon_allow_pool_delete": "true",

#重试删除pool成功
$ ceph osd pool delete userFeedPool userFeedPool --yes-i-really-really-mean-it
pool 'userFeedPool' removed

#下面错误也同样消失
too many PGs per OSD
#引起的原因： 集群osd 数量较少，搭建rgw网关、OpenStack、容器组件等，pool创建较多，每个pool默认需要占用一些pg，pool中pg数目设置不合理，导致集群 total pg 数过多

3.2. 归置组

Placement Groups：
一个归置组(PG)把一系列对象汇聚到一组，并且把这个组映射到一系列OSD。跟踪每个对象的位置和元数据需要大量计算。例如，一个拥有数百万对象的系统，不可能在每个对象级追踪位置。归置组可应对这个影响性能和扩展性的问题，另外，归置组减小了ceph存储、检索数据时必须追踪的每个对象元数据的处理量和尺寸。

设置PG数目（pgp_num应该设成和pg_num一样）

Total PGs = (Total_number_of_OSD * 100) / max_replication_count

所以3*100/2=150，Ceph官方推荐取最接近2的指数倍，所以选择128，另外要考虑之后的扩容需求。

pg-num 与 pgp-num 只可以扩大不可以缩小
pg-num 与 pgp-num 常见数量一样
建议在创建池时候就定义好合理的 pg-num 与 pgp-num 因为在改变(增加) pg-num 与 pgp-num 时候, 会自动产生 rebalance 操作, 会严重影响 ceph cluster io 性能

#处理PG数目不对的警告
$ ceph health
HEALTH_WARN too few PGs per OSD (16 < min 30)
$ ceph osd pool get cephfs_data min_size
min_size: 2
$ ceph osd pool get cephfs_data pg_num
pg_num: 8
$ ceph osd pool set cephfs_data pg_num 128
Error E2BIG: specified pg_num 128 is too large (creating 120 new PGs on ~3 OSDs exceeds per-OSD max of 32)
$ ceph osd pool set cephfs_data pg_num 64
set pool 1 pg_num to 64
$ ceph health
HEALTH_WARN 1 pools have pg_num > pgp_num
$ ceph osd pool set cephfs_metadata pg_num 64
set pool 2 pg_num to 64
$ ceph health
HEALTH_WARN 2 pools have pg_num > pgp_num
$ ceph osd pool set cephfs_metadata pgp_num 64
set pool 2 pgp_num to 64
$ ceph osd pool set cephfs_data pgp_num 64
set pool 1 pgp_num to 64
$ ceph health
HEALTH_OK
